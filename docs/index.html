AgentSuite v1.0.12

Turn a business goal into 26 structured artifacts -- brand system, product spec, architecture, marketing plan, risk register -- in one command.

For technical founders and developers building in Claude Code, Cowork, Codex, or Google Antigravity.

View on GitHub

A real agentsuite founder run end-to-end: five stages, 14 artifacts, cost summary. Browse the full output directory.

Install

pip install "agentsuite[anthropic] @ git+https://github.com/scottconverse/AgentSuite.git"
# other providers: [openai]  [gemini]  [ollama]  [all]

Or run without installing via uvx (MCP mode):

uvx --from "agentsuite[mcp] @ git+https://github.com/scottconverse/AgentSuite.git" agentsuite-mcp

Single agent

export ANTHROPIC_API_KEY=sk-ant-...

agentsuite founder run \
  --business-goal "Launch PatentForgeLocal v1" \
  --project-slug pfl

# Review output, then approve to promote to _kernel/pfl/
agentsuite founder approve --latest --project-slug pfl --approver you

Chain agents (pipeline)

Run multiple agents back-to-back. Each agent's approved output feeds the next via _kernel/<slug>/.

# Headless -- auto-approves each step
agentsuite pipeline run \
  --agents founder,design,product \
  --project-slug pfl \
  --business-goal "Launch PatentForgeLocal v1" \
  --auto-approve

# With review gates (default) -- pauses after each agent
agentsuite pipeline run \
  --agents founder,design,product \
  --project-slug pfl \
  --business-goal "Launch PatentForgeLocal v1"

# Prints artifact list + approve command. When ready:
agentsuite pipeline approve --pipeline-id <id>

# Lost the pipeline ID? List all pipelines:
agentsuite pipeline list

MCP (Claude Code / Cowork)

Add to .mcp.json in your project root:

{
  "mcpServers": {
    "agentsuite": {
      "command": "uvx",
      "args": ["agentsuite-mcp"],
      "env": {"AGENTSUITE_ENABLED_AGENTS": "founder,design,product,engineering,marketing,trust_risk,cio"}
    }
  }
}

MCP (Codex)

Add to ~/.codex/mcp.toml:

[servers.agentsuite]
command = "uvx"
args = ["agentsuite-mcp"]

[servers.agentsuite.env]
AGENTSUITE_ENABLED_AGENTS = "founder,design,product,engineering,marketing,trust_risk,cio"

Google Antigravity: MCP-compatible. Use the same uvx agentsuite-mcp command with AGENTSUITE_ENABLED_AGENTS set. See the Antigravity docs for the config format.

What each agent produces

Founder 14 artifacts

Brand system, brief library, voice guide, prompt library, QA report.

Design 14 artifacts

Visual identity, design system, component spec, brand QA scoring.

Product 14 artifacts

PM intent to UI spec to coding handoff, user story map, acceptance criteria.

Engineering 14 artifacts

Architecture decisions, API spec, data model, security review, deployment plan, runbook, tech-debt register.

Marketing 14 artifacts

Campaign brief, audience profile, messaging framework, content calendar, channel strategy, SEO plan, launch plan.

Trust / Risk 14 artifacts

Threat model, risk register, control framework, incident response plan, compliance matrix, audit readiness report.

CIO 14 artifacts

IT strategy, technology roadmap, digital transformation plan, enterprise architecture, budget model, workforce plan.

What stays between sessions

Approved artifacts are promoted to _kernel/<slug>/ and picked up automatically by the next agent or session.
