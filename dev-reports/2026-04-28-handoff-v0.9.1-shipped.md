## What shipped (v0.9.0 surface, v0.9.1 working pipeline)

All seven Sprint 3 items from `dev-reports/2026-04-28-sprint-plan-to-v1.0.md`:

1. **Per-run cost telemetry** — `cost_summary.json` with per-stage breakdown, identity fields, schema pinned by tests. ADR-0005 documents the cap-vs-telemetry split.
2. **RunState subclass-aware persistence** — `_state.json` envelope now carries `schema_version: 2`. Subclass-specific input fields survive save/load round-trip. Pre-v0.9 state raises `RunStateSchemaVersionError`. ADR-0002 records the decision.
3. **Golden test content assertions (lite)** — `tests/golden/_helpers.py` with `assert_artifact_exact()` + `assert_qa_within_tolerance(rtol=0.05)`. Founder snapshot extended with `brand-system.md` + `qa_scores.json`. Remaining 6 agents deferred to v0.9.2.
4. **CIO `as_of_date` field** + tz-aware date helpers. Two runs with different dates produce different artifact strings.
5. **Resume-from-failure idempotency** — `_drive()` carries `state.cost_so_far` across resume + rehydrates per-stage. ADR-0007 + integration test pin the contract.
6. **ADR backfill** — 7 ADRs under `docs/adr/` covering rubric, RunState, retry, MCP naming, cost split, no-PyPI, idempotency. CONTRIBUTING points at the index.
7. **Clean-install verification on tag push** — README install markers, drift check, smoke step running `agentsuite --help` + `agentsuite-mcp --help` against the wheel install.

Plus: v0.8.3 (security hygiene: pip-audit + SBOM + provider drift), v0.8.4 (release.yml hotfix for tool paths). Also pre-existing bugs fixed inline along the way (CRLF leak in verify-release.sh, CHANGELOG link typo, cost carry-forward bug, missing `[mcp]` extra in smoke step).
